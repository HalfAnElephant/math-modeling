# 论文同步与最终验收 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` or `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将所有子计划生成的结果同步到论文正文、图表和 Word 成稿，确保最终交付不再包含旧结论、无支撑结论或与 CSV/JSON 不一致的数值。

**Architecture:** 先重新生成所有实验和图表，再逐节更新 `report/c_uav_inspection_paper.md`，最后用 `build_paper.py` 生成 `C题论文_多无人机联合巡检优化.docx`。所有数值以 `outputs/c_uav_inspection/` 为唯一来源。

**Tech Stack:** Python 3.11, existing experiment runner, plot generator, Pandoc via `build_paper.py`, shell `rg`.

```markdown
## 5.7 算法复杂度分析

设目标数为 $n$，生成航次数为 $R$，单条航次最多访问 $m_r$ 个目标，人工复核点数为 $m=|M|$。最近邻初始序列复杂度为 $O(n^2)$。可拆分和不可拆分悬停装箱均为 $O(Rn)$。2-opt 对航次 $r$ 的复杂度可写为 $O(I_r m_r^3)$。LPT 分配复杂度为 $O(R\log R+RK)$。Held-Karp 地面 TSP 时间复杂度为 $O(m^2 2^m)$，空间复杂度为 $O(m2^m)$。重建搜索最多评价 $n$ 个候选，复杂度约为 $O(n(C_{\mathrm{uav}}+m^2 2^m))$。全枚举验证遍历 $2^n$ 个直接确认集合，复杂度约为 $O(2^n(C_{\mathrm{uav}}+m^2 2^m))$；本文 $n=16$，枚举规模为65536，可用于赛题规模验证。
```
