# C题论文修改子计划总览

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` or `superpowers:executing-plans` to implement these subplans task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将总计划拆分为可独立执行、可测试、可验收的子计划，使实现者可以按顺序完成全部论文与实验修正。

**Architecture:** 子计划按依赖顺序排列。先修正问题一时间优先求解，再修正问题二优先级建模，随后补实验对照、枚举验证、敏感性分析，最后统一更新论文与 Word 成稿。

**Tech Stack:** Python 3.11, `openpyxl`, `pytest`, `matplotlib`, standard library `dataclasses`, `itertools`, `csv`, `json`; project root is `/Users/xcy/Documents/100 项目/数学建模`.

## 1. 子计划执行顺序

必须按下列顺序执行。原因是前序计划会改变后序计划使用的核心结果。

| 顺序 | 子计划文件 | 目标 |
|---:|---|---|
| 1 | `01_问题一时间优先与K4复核.md` | 验证并修正问题一“为什么不是 K=4 最短”的核心缺陷 |
| 2 | `02_问题二优先级权重建模.md` | 将 `priority_weight` 正式纳入问题2模型、评分和接受准则 |
| 3 | `03_基准对比与换电敏感性.md` | 补“仅基础巡检 vs 联合优化”基准，并把换电敏感性改到 `K=1` |
| 4 | `04_可拆分悬停对照实验.md` | 补可拆分与不可拆分悬停的量化对照 |
| 5 | `05_重建搜索全枚举验证.md` | 对 16 个目标直接确认集合做全枚举或等价验证 |
| 6 | `06_接受准则与能量参数敏感性.md` | 补 `1.03` 接受准则、电量、悬停功率敏感性 |
| 7 | `07_论文同步与最终验收.md` | 将所有结果写回论文、重建图表和 docx，并做最终验收 |

## 2. 全局约束

- 所有代码改动必须保持现有公开 API 的默认行为尽量兼容。
- 任何会改变推荐方案的实验，必须重新生成 `outputs/c_uav_inspection/` 下结果文件。
- 论文正文只编辑 `report/c_uav_inspection_paper.md`，不要手工编辑 `C题论文_多无人机联合巡检优化.docx`。
- Word 成稿只能通过 `.venv/bin/python build_paper.py` 生成。
- 如果某个实验结果与原论文数值不同，以重新生成的 CSV/JSON 为准。
- 不允许在论文中写未经实验支撑的结论，例如“只需2架无人机”或“重建搜索近似最优”。

## 3. 全局命令

```bash
cd "/Users/xcy/Documents/100 项目/数学建模"
```

```bash
.venv/bin/python -m pytest tests -q
```

```bash
.venv/bin/python - <<'PY'

from c_uav_inspection.experiments import run_all_experiments
from c_uav_inspection.plots import generate_all_figures

data_path = Path("2026同济数学建模竞赛赛题/2026C数据.xlsx")
out = Path("outputs/c_uav_inspection")
run_all_experiments(data_path, out)
generate_all_figures(data_path, out)
PY
```

重新生成 Word：

```bash
.venv/bin/python build_paper.py
```

## 5. 推荐提交节奏

每完成一个子计划并通过对应测试后提交一次：

```bash
git add <子计划涉及文件>
git commit -m "fix: <简短说明>"
```

建议提交顺序：

1. `fix: validate problem1 time-priority routing`
2. `fix: include priority weights in problem2`
3. `fix: add baseline and swap sensitivity experiments`
4. `fix: add split hover ablation`
5. `fix: add direct-set enumeration validation`
6. `fix: add parameter sensitivity experiments`
7. `docs: update paper with revised results`
