### 3.1 根因说明

- [ ] **Step 1: 在计划执行记录中明确旧算法偏差**

当前问题一管线是：

```text
nearest_neighbor_order
-> split_order_into_energy_feasible_routes
-> improve_route_by_two_opt
-> LPT assign routes to K UAVs
```

其中 `split_order_into_energy_feasible_routes` 的行为是：沿一个固定目标顺序尽量把当前航次的能量用满，只有塞不下时才开下一趟。它优化的是“能耗可行装箱”和“较少航次数”，不是直接优化：

$$
\min T_u=\min \max_{u\in U} W_u.
$$

因此当前 `K=4` 仍只有2趟可分配，不代表题目最优；它只代表“旧装箱策略生成了2趟”。

### 3.3 候选航次预计算

- [ ] **Step 3: 实现 `precompute_problem1_subset_routes`**

函数签名：

```python
def precompute_problem1_subset_routes(
    data: ProblemData,
    improve: bool = True,
) -> dict[int, SubsetRouteCandidate]:
```

实现要求：

1. 令 16 个目标映射为 bitmask 的 16 位。
2. 枚举所有非空子集，最多 `2^16-1=65535` 个。
3. 对每个子集，用 `nearest_neighbor_order(data, subset_node_ids)` 构造访问顺序。
4. 悬停时间使用完整基础需求：

```python
hover_times = {node_id: target_by_id[node_id].base_hover_time_s for node_id in subset_node_ids}
```

5. 构造单趟路线：

```python
route = UAVRoute(
    uav_id=0,
    sortie_id=1,
    node_sequence=(0,) + order + (0,),
    hover_times_s=hover_times,
)
```

6. 若 `improve=True`，调用 `improve_route_by_two_opt(route, data)`。
7. 调用 `evaluate_uav_route`。
8. 只保留 `metrics.feasible_energy == True` 的候选。

注意：这个求解器用于时间优先验证，采用“一目标完整基础悬停归属某一并行航次”的划分模型。它不替代可拆分悬停装箱，而是检验在题目问题一“优先考虑时间”的语境下，主动增加并行航次是否更优。

### 3.4 K 路划分动态规划

- [ ] **Step 4: 实现 `solve_problem1_time_priority_for_k`**

函数签名：

```python
def solve_problem1_time_priority_for_k(
    data: ProblemData,
    k: int,
    improve: bool = True,
) -> TimePriorityProblem1Solution:
```

动态规划目标：在最多 `k` 条并行航次中覆盖全部目标，使最大单航次时长最小。

定义：

```python
full_mask = (1 << n) - 1
best_max[r][mask] = 使用 r 条航次覆盖 mask 的最小最大航次时长
parent[r][mask] = 最后一条航次使用的 subset mask
```

递推：

```python
best_max[0][0] = 0.0
best_max[0][mask != 0] = inf

for r in range(1, k + 1):
    for mask in range(0, full_mask + 1):
        sub = mask
        while sub:
            if sub in candidates:
                candidate = max(best_max[r - 1][mask ^ sub], candidates[sub].duration_s)
                if candidate < best_max[r][mask]:
                    best_max[r][mask] = candidate
                    parent[r][mask] = sub
            sub = (sub - 1) & mask
```

复杂度：`O(K*3^n)`，本文 `K<=4,n=16`，用于问题一复核可接受。

重构路线：

1. 从 `best r in 1..k` 中选择 `best_max[r][full_mask]` 最小者。
2. 反向读取 `parent` 得到若干子集。
3. 将每条 route 的 `uav_id` 设为 `1..route_count`，`sortie_id=1`。
4. 调用 `summarize_uav_solution(routes, data, battery_swap_time_s=data.params.battery_swap_time_s)`。
5. 验证每个目标累计悬停恰好等于 `base_hover_time_s`。

若 `k=1` 且全目标单趟不可行，则该 DP 会不可行。此时 `solve_problem1_for_k` 可继续使用旧可拆分多趟方案作为 `K=1` 结果，但论文必须说明 `K=1` 需要多趟和换电。

- [ ] **Step 4: 修改重建搜索接受准则**

当前逻辑：


改为：

```python
    accepted = True

    new_weighted < cur_weighted
    and new_closed_time <= cur_closed_time * 1.03 + 1e-9

    new_manual < cur_manual
```

判定顺序说明：

\left(T'<T\right)

\left(C_M'<C_M,\ T'\le \rho T\right)

\left(|M'|<|M|,\ T'\le \rho T\right),
$$

其中 $\rho=1.03$，并在后续敏感性分析中取 $1.00,1.03,1.05,1.10$。

新增函数：

```python
def split_order_into_energy_feasible_routes_no_split(
    order: tuple[int, ...],
    hover_times_s: Mapping[int, float],
    data: ProblemData,
) -> tuple[UAVRoute, ...]:
```

约束：

1. 对每个正需求目标 `i`，只能在某一趟中完整服务 `hover_times_s[i]`。
2. 每个目标在 `hover_times_s` 中的正需求只能出现在一个 route 的 `hover_times_s` 中。
3. 如果当前航次加入完整目标会导致“到达 + 完整悬停 + 返回”超过能耗限制，则结束当前航次并新开一趟。
4. 如果单目标独立航次也不可行，抛出 `InfeasibleError`。

核心逻辑结构：

```python
for node_id in order:
    demand_s = remaining_hover[node_id]
    if demand_s <= EPSILON:
        continue

    full_service_energy = (
        data.flight_energy_j[(current_node, node_id)]
        + demand_s * hover_power
        + data.flight_energy_j[(node_id, 0)]
    )

    if energy_used_j + full_service_energy > limit_j + EPSILON:
        close_current_sortie()
        start_new_sortie()

    if single_node_energy(node_id, demand_s) > limit_j + EPSILON:
        raise InfeasibleError(...)

    append_full_node_service(node_id, demand_s)
```

注意：不要改变现有 `split_order_into_energy_feasible_routes` 默认行为，防止已有测试回归。

```markdown
## 5.7 算法复杂度分析

设目标数为 $n$，生成航次数为 $R$，单条航次最多访问 $m_r$ 个目标，人工复核点数为 $m=|M|$。最近邻初始序列每轮从未访问目标中选最短飞行时间，复杂度为 $O(n^2)$。可拆分悬停装箱在每一趟中最多扫描一次目标序列，复杂度为 $O(Rn)$；不可拆分对照算法同阶。2-opt 对航次 $r$ 每轮尝试 $O(m_r^2)$ 个片段，每次评价路线需 $O(m_r)$，若迭代轮数为 $I_r$，复杂度为 $O(I_r m_r^3)$。LPT 多机分配先按航次工作量排序，再逐趟选择当前最空闲无人机，复杂度为 $O(R\log R+RK)$。物业人员路径采用 Held-Karp 动态规划，时间复杂度为 $O(m^2 2^m)$，空间复杂度为 $O(m2^m)$。重建搜索最多评价 $n$ 个候选直接确认目标，每次包含一次无人机重规划和一次地面 TSP，因此复杂度约为 $O(n(C_{\mathrm{uav}}+m^2 2^m))$。全枚举验证遍历 $2^n$ 个直接确认集合，复杂度约为 $O(2^n(C_{\mathrm{uav}}+m^2 2^m))$；本文 $n=16$，枚举规模为 65536，仍可在小规模赛题数据上接受。
```
