# 实验体系重跑与结果文件治理 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 建立可复现实验输出体系，重跑所有受数据和模型修复影响的结果，清理或归档过期文件。

**Architecture:** `experiments.py` 是结果文件唯一生成入口，`plots.py` 是图表唯一生成入口。输出目录增加 manifest，标明每个文件的生成命令、是否昂贵、是否论文引用。

**Tech Stack:** Python 3.11, CSV/JSON, `matplotlib`, `pytest`, file manifest.

- [ ] **Step 2: 归档过期输出**

不要直接删除旧文件。新增目录：

```text
outputs/c_uav_inspection/archive_before_review_remediation/
```

将不再使用的旧输出移入归档，例如：

```text
problem1_k_comparison.csv
problem1_swap_sensitivity.csv
```

归档命令示例：

```bash
mkdir -p outputs/c_uav_inspection/archive_before_review_remediation
mv outputs/c_uav_inspection/problem1_k_comparison.csv \
   outputs/c_uav_inspection/archive_before_review_remediation/
```

若文件不存在，记录在 manifest 的 notes 中，不视为失败。
