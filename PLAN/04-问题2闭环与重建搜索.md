## 2. 实现闭环评价

创建 `c_uav_inspection/problem2.py`，必须包含：

- `GroundReviewResult`
  - `path`
  - `travel_time_s`
  - `service_time_s`
  - `total_time_s`

- `ClosedLoopResult`
  - `direct_confirmed_nodes`
  - `manual_nodes`
  - `manual_count`
  - `uav_phase_time_s`
  - `ground_review_time_s`
  - `closed_loop_time_s`
  - `ground_path`

- `effective_direct_threshold(target, direct_threshold_multiplier)`
  - multiplier 必须大于 0。
  - 返回 `max(target.base_hover_time_s, target.direct_confirm_time_s * direct_threshold_multiplier)`。

- `solve_ground_tsp(data, manual_point_ids)`
  - 输入是物业点 ID 元组，如 `("MP01", "MP03")`。
  - 若为空，返回路径 `("P0", "P0")`，总时间为 0。
  - 使用动态规划或带缓存枚举求最短地面通行时间。
  - 服务时间为所有待复核点 `manual_service_time_s` 之和。

- `evaluate_closed_loop(data, routes, direct_threshold_multiplier)`
  - 先统计每个目标累计悬停。
  - 累计悬停达到有效阈值的目标进入 `direct_confirmed_nodes`。
  - 其他目标进入 `manual_nodes` 和物业 TSP。
  - `closed_loop_time_s = uav_phase_time_s + ground_review_time_s`。
  - `uav_phase_time_s` 使用 `summarize_uav_solution(routes, data, data.params.battery_swap_time_s)`。

```python
from c_uav_inspection.problem2 import solve_joint_problem_for_k


def test_joint_solver_reduces_or_matches_manual_count_against_base_only():
    data = load_problem_data(DATA_PATH)
    base = evaluate_closed_loop(data, solve_problem1_for_k(data, k=3, battery_swap_time_s=300).routes, 1.0)
    joint = solve_joint_problem_for_k(data, k=3, direct_threshold_multiplier=1.0)

    assert joint.closed_loop.manual_count <= base.manual_count
    assert joint.closed_loop.closed_loop_time_s > 0
    assert all(route.node_sequence[0] == 0 and route.node_sequence[-1] == 0 for route in joint.routes)


def test_joint_solver_direct_confirmed_nodes_meet_effective_thresholds():
    data = load_problem_data(DATA_PATH)
    joint = solve_joint_problem_for_k(data, k=4, direct_threshold_multiplier=0.70)
    hover_by_node = {}
    for route in joint.routes:
        for node_id, seconds in route.hover_times_s.items():
            hover_by_node[node_id] = hover_by_node.get(node_id, 0.0) + seconds

    by_id = {target.node_id: target for target in data.targets}
    for node_id in joint.closed_loop.direct_confirmed_nodes:
        assert hover_by_node[node_id] >= effective_direct_threshold(by_id[node_id], 0.70)
```

## 4. 实现直接确认集合重建搜索

在 `c_uav_inspection/problem2.py` 追加：

- `JointSolution`
  - `routes`
  - `closed_loop`

- `_direct_confirm_score(data, target_id, direct_threshold_multiplier)`
  - 使用有效阈值计算额外悬停。
  - 使用 `P0` 往返该物业点 + 服务时间作为人工节省近似。
  - 额外能耗惩罚为直接确认往返能耗占单趟能耗上限的比例。

- `_hover_requirements_for_direct_set(data, direct_nodes, direct_threshold_multiplier)`
  - 直接确认集合中的目标需求为有效阈值。
  - 其他目标需求为基础悬停时间。

- `_rebuild_for_direct_set(data, k, direct_nodes, direct_threshold_multiplier)`
  - 调用 `solve_uav_hover_plan(..., improve=True)` 重新规划全部无人机路线。
  - 若任何单趟能耗不可行或 `uav_phase_time_s` 超过运营时长，返回 `None`。
  - 否则返回 `JointSolution`。

- `solve_joint_problem_for_k(data, k, direct_threshold_multiplier)`
  - 先用空直接确认集合得到基础方案。
  - 按 `_direct_confirm_score` 从高到低尝试加入候选目标。
  - 每加入一个目标后，必须整体重建路线。
  - 接受条件：
    - 闭环总时间下降；或
    - 人工复核点数减少，且闭环总时间不超过当前最优方案的 `1.03` 倍。
  - 返回最终 `JointSolution`。

明确禁止的实现方式：

```text
不要在某点超限时直接新建单点航次作为主策略。
不要只修改原航次悬停时间而不重建其他航次。
```
